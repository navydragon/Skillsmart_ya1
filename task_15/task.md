15. Event Sourcing
Тут кто-нибудь ещё есть? Если вы добрались досюда, то вы большой молодец!

Event Sourcing -- это, проще говоря, построение состояния на основе списка прошлых событий. Мы получаем по сути систему с детерминированными состояниями, причём её можно откатывать и возвращать с помощью undo/redo, как в машине времени.

Клиент отправляет команду в CommandHandler, который перед её обработкой перестраивает текущее состояние с нуля с помощью доступа к Event Store, на основе некоторого набора считанных из него последних событий, связаных с конкретным роботом.

Далее CommandHandler проверяет команду и решает как её выполнить с учётом текущего пересозданного состояния. При этом формируется (возможно пустой) список событий, который записывается в Event Store.

Фишка в том что ни клиенту, ни обработчику команд не нужно отслеживать состояние на протяжении времени. Мутабельное только EventStore!
