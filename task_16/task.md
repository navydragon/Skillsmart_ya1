### **16. Stream Processing**

Stream Processing -- это когда "события" текут через систему, любой из нескольких процессоров их обрабатывает и может генерировать новые.


```javascript
Command Handler при этом становится "тупым":
- только валидация команды
- генерация событий
- сохранение в EventStore
- никакой бизнес-логики

Но появляются Event Processors/Aggregators:
- подписываются на события из EventStore
- выполняют всю сложную бизнес-логику
- могут генерировать новые события
- работают асинхронно

Преимущества:
- разделение ответственности
- масштабируемость (процессоры независимы)
- легко добавлять новую логику через новые процессоры
- Command Handler остаётся простым
```

 

  

### **Stream Processing: задание**

Итак, вкратце:

```javascript
CommandHandler стал "тупым" - только генерирует события из команд.
События разделяются на запросы (MoveRequestedEvent) и результаты (RobotMovedEvent).
EventStore с подписками уведомляет процессоры о новых событиях.
EventProcessor'ы (агрегаторы) выполняют всю бизнес-логику и генерируют события результатов.
```

В такой архитектуре будет легко добавлять новую логику через новые процессоры, а CommandHandler остаётся простым.
